Document Preprocessing

Before clustering, each document underwent preprocessing to standardize the format and structure of the text data. This included steps such as:

  1. Tokenization:

Breaking down the text into individual words or tokens.

  • Sentence Tokenization: Each document was split into sentences using NLTK's sent_tokenize function.
  • Word Tokenization: Each sentence was further tokenized into words using NLTK's word_tokenize function.
  1. Filtering Tokens

After tokenization, the tokens were filtered to remove irrelevant words and noise from the text data. The filtering process involved the following steps:

  • Punctuation Removal: Punctuation, special characters, and newline characters were removed from the tokens using regular expressions (re-module).
  • Additional Conditions: Tokens containing URLs, email addresses, HTML tags, and non-alphabetic characters were excluded from the final token list.
  • Stopword Removal: Using a generic stopword list may not be optimal for all datasets, especially when dealing with domain-specific or specialized content.

    To address this limitation and enhance the quality of text preprocessing, a custom stopwords list was generated based on the TF-IDF scores of terms in the document corpus.

    By calculating the average TF-IDF scores for each term across the corpus, terms more representative of the document topics can be identified.

  1. Token Stemming and Lemmatization: To reduce tokens to their base or dictionary form
  • Stemming: Snowball Stemmer from NLTK was used to stem tokens, reducing them to their root or base form. This process helps in consolidating variations of words with the same meaning.
  • Lemmatisation: WordNet Lemmatiser from NLTK was employed to lemmatise stemmed tokens, ensuring that tokens were transformed into valid words in the English language.

The preprocessed tokens, after filtering, stemming, and lemmatisation, constituted the final token list used for feature extraction and subsequent clustering analysis.

1. Observations upon Analysis

Random initialisation trap - effect on Reproducibility and Stabilisation

The PCA plots and Clusters were different with each iteration hence, the following changes were made to keep the model’s processes as deterministic as possible:

  • Vector Embedding was changed from Doc2Vec to TF-IDF vectorisation. Doc2Vec model in Gensim can exhibit non-deterministic behaviour due to random initialisation
  • Shuffling/randomisation using a seed value for required processes was done. Using the same random seed ensures that the randomness does not introduce variability into any process. This ensured reproducibility with every run, since the pseudo-random generator generated the same sequence of numbers hence, consistent results.

Hence, PCA was performed before K-Means clustering; After performing TF-IDF vectorisation on all documents, the vectors were normalised and the dimensions were reduced from 3761 to 2.

Instead of randomly initialising the centroids for K-Means clustering, the range of the PCA-transformed data along each principal component is calculated, and K evenly spaced points are generated within this range

To prevent overfitting of the model, the order of the documents loaded from the corpus is shuffled with a random seed. This prevents any bias from the ordered documents.

  • Number of iterations for centroid initialisation: 2000
  • Number of iterations for K-Means fitting: 10,000
  • Tolerance: 1e-4

number of iterations was increased, and tolerance decreased, to ensure good stabilisation

Choropleth Layer Visualisation

The choropleth method in Folium creates a choropleth layer by mapping data values to colors for each geographic feature (in this case, each district in San Francisco).

The GeoJSON file contains the geographic boundaries of the San Francisco neighbourhoods (districts). Each feature in this file represents a district and includes properties like the district name or identifier.

Folium combines the geographic features from the GeoJSON file with the data from the DataFrame to render the choropleth layer. Each district is filled with the appropriate colour based on its crime count.

  • The DataFrame contains aggregated crime data by district, not individual crime locations.
  • GeoJSON Mapping: The GeoJSON file provides district boundaries, and the choropleth method maps these boundaries to the aggregated crime data.

Plotly and Mapbox instead of folium

  • Mapbox uses WebGL for rendering, which leverages the GPU for better performance with large-scale geographical data.
  • Folium is more memory-intensive for large datasets. Each marker is added individually to the map, leading to increased processing time and memory usage.

HDBSCAN - Hierarchical Density-Based Spatial Clustering of Applications with Noise

HDBSCAN is an extension of DBSCAN that is more efficient and scalable for large datasets; it performs DBSCAN over varying epsilon values and integrates the result to find a clustering that gives the best stability over epsilon. This allows HDBSCAN to find clusters of varying densities (unlike DBSCAN), and be more robust to parameter selection.

In practice this means that HDBSCAN returns a good clustering straight away with little or no parameter tuning -- and the primary parameter, minimum cluster size, is intuitive and easy to select.

Final Conclusion

  • There was very little correlation between the provided features and the crime categories
  • Meaningful patterns/relationships could not be extrapolated from yearly demographic data and the crime trends either.
  • Clubbing similar categories together contributed to increased accuracy owing to less sparsity. However, converting time features into cyclic representations or attempting to balance the dataset using a combination of over-sampling and under-sampling techniques leads to reduced accuracies and overfitting.
  • Most times, the models ended up memorising the training data and overfitting to it- suggested by the consistent and large differences between training and validation accuracies.
  • Almost all categories of crime were misclassified as property crimes, which happened to be the category with most amount of samples.
  • LSTM and a more complex Neural Network architecture (with 2 hidden layers) confused administrative and property crimes. Whereas, a simpler neural network confused property and violent crimes the most.
  • The best results were obtained using traditional ensemble models XGBoost and Random Forest on clubbed categories - the variance was lower and the validation accuracy was higher (0.4336)

Future Prospects

1. Explainable Artificial Intelligence (XAI)

The integration of Explainable AI (XAI) in crime prediction models presents significant potential. XAI aims to make the decision-making processes of AI systems’ black boxes transparent and understandable to human users. By adopting XAI, law enforcement agencies can:

  • Enhance Trust: Providing clear insights into how predictions are made increases trust among stakeholders.
  • Improve Decision-Making: Understanding the factors influencing predictions can help in better decision-making and policy formulation.
  • Ensure Accountability: Transparency in AI decisions ensures that systems remain accountable and bias is minimized.

2. State-of-the-Art Models: STHSL

State-of-the-art models like Self-supervised Hypergraph Learning (STHSL) hold great promise in the realm of crime prediction. STHSL leverages hypergraph-based self-supervised learning methods to address the challenges of label scarcity in crime datasets.

  • Handling Label Scarcity: STHSL can effectively tackle the issue of label scarcity by using self-supervised learning methods as auxiliary tasks. This allows the model to learn from the inherent structure of the data without relying heavily on labeled instances.
  • Enhanced Predictive Accuracy: By incorporating the relationships between different crime instances in a hypergraph structure, STHSL can capture complex dependencies, leading to improved predictive accuracy.
  • Robust Learning: The self-supervised learning approach enables STHSL to learn robust representations, which are crucial for making reliable predictions in real-world scenarios.

3. Future Research Directions

  • Enhancing Model Interpretability: Further research can focus on developing more advanced XAI techniques specifically tailored for hypergraph-based models.
  • Combining Multiple Data Sources: Integrating data from various sources such as social media, economic indicators, and environmental factors can provide a more comprehensive view of crime patterns.
  • Real-Time Crime Prediction: Developing systems capable of real-time crime prediction and alerting can significantly improve the responsiveness of law enforcement agencies.
